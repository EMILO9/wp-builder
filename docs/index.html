Quickstart

Start fast — three steps to a packaged plugin.

  1. Install (dev dependency): 
    npm install --save-dev wp-builder
  2. Create builder.config.ts with minimal fields: 
    export default {
  header: { pluginName: 'My Plugin' },
  build: { copy: ['readme.txt', 'LICENSE', 'assets/**'] }
}
  3. Run the build: 
    npx wp-builder build

After the run you'll have a staging folder at .plugin/<slug>/ (slug derived from your plugin name) which you can zip for distribution.

Configuration

Primary fields you should know about.

Example shape

interface BuildConfig {
  header: { pluginName: string }
  build?: { copy?: string[]; entry?: Record }
}

User-facing docs (copy)

/**
 * Files or glob patterns to include in the plugin package.
 * Patterns are evaluated from the project root.
 * Matched files are copied into the plugin staging folder preserving their paths.
 * Examples: ["readme.txt", "LICENSE", "assets/screenshots/**"]
 * @default []
 */
copy: string[];

Copy patterns & behavior

This section explains what happens when you add patterns to build.copy.

How patterns are evaluated

  • Patterns run with fast-glob from process.cwd().
  • Options: onlyFiles: true, dot: true, absolute: true.
  • Staging folder is ignored (computed from ctx.stagingPath and normalized to POSIX slashes).
  • Matches are deduplicated, so overlapping patterns don't copy a file multiple times.

Examples

"a/**"   // all files under directory a (includes a/b/x.txt)
"a/*"    // files immediately under a (no recursion)
"**/*"   // everything in the repo (use carefully)

Outside-repo files

If a pattern matches files outside the repo root, the current implementation will copy them into the plugin root using their basename (e.g. /tmp/foo.png -> .plugin/<slug>/foo.png).

Tip: avoid matching outside the repo to prevent accidental collisions. If you'd rather skip outside files, the task can be adjusted — tell me and I can change it.

Important internal tasks

prepare_runtime_context

ctx.slug = kebabCase(config.header.pluginName)
ctx.buildRoot = path.join(process.cwd(), '.plugin')
ctx.stagingPath = path.join(ctx.buildRoot, ctx.slug)

copy_assets

Copies files matched by build.copy into the staging folder while preserving repo-relative paths and skipping the staging output.

package_plugin_archive

Archives the staging folder into a ZIP suitable for distribution.

FAQ & Troubleshooting

My files aren't being copied

  • Ensure patterns are relative to project root (no leading slash).
  • Check staging folder ignore — files under .plugin/ are excluded.
  • Try a narrower pattern to confirm matches.

How to include empty directories?

The current copy task uses onlyFiles: true, so empty folders aren't preserved. If you need them, the task can be modified to create directories explicitly.
